# Data Model: GitHub Release Action

**Phase**: 1 — Design  
**Feature**: 026-github-release-action  
**Date**: 2026-03-31

---

## Context

This feature is a CI/CD workflow, not an in-app data feature. There are no new Room entities, database tables, or in-app model classes introduced. The entities below represent the conceptual objects that exist within the GitHub Actions release pipeline and the GitHub Releases system.

---

## Entities

### ReleaseTrigger

The event that initiates the release pipeline run.

| Field | Type | Description |
|-------|------|-------------|
| `kind` | enum (`workflow_run`, `workflow_dispatch`) | How the release was initiated |
| `headSha` | String | The git commit SHA being released |
| `sourceBranch` | String | Branch from which the release is cut (`main` or other) |
| `isPrerelease` | Boolean | `true` if branch ≠ `main` or manual prerelease input is `true` |

**State transitions**: Trigger is immutable once the pipeline starts.

---

### ReleaseVersion

The version state read from `version.properties` after the Gradle build completes.

| Field | Type | Source | Description |
|-------|------|--------|-------------|
| `versionName` | String | `version.properties` (`versionName`) | Semver string, e.g. `0.12.4` |
| `versionCode` | Int | `version.properties` (`versionCode`) | Monotonically increasing integer |
| `tagName` | String | Derived: `v{versionName}` | Git tag to apply, e.g. `v0.12.4` |
| `apkFileName` | String | Derived: `ndi-for-android-{versionName}.apk` | Expected APK filename after Gradle rename |
| `apkPath` | String | Derived: `app/build/outputs/apk/release/{apkFileName}` | Deterministic local workspace path |

**Validation rules**:
- `versionName` MUST match pattern `\d+\.\d+\.\d+`
- `versionCode` MUST be greater than zero
- `apkPath` file MUST exist and have size > 0 before the release step runs

---

### ReleaseArtifact (APK)

The shipment-ready Android Package produced by the Gradle `assembleRelease` task.

| Field | Type | Description |
|-------|------|-------------|
| `localPath` | String | Absolute workspace path to the APK |
| `fileName` | String | `ndi-for-android-{versionName}.apk` |
| `sizeBytes` | Long | File size; must be > 0 (verified before upload) |
| `buildType` | String | Always `release` for this workflow |
| `signing` | String | `debug` (initial scope; keystore signing deferred) |

---

### GitHubRelease

The release entry published to the repository's Releases page.

| Field | Type | Description |
|-------|------|-------------|
| `tagName` | String | `v{versionName}` — must be unique within the repo |
| `releaseName` | String | `NDI for Android {versionName}` |
| `body` | String | Auto-generated by GitHub from PRs/commits since previous tag |
| `prerelease` | Boolean | `true` for non-main-branch manual triggers |
| `assets` | List<String> | Exactly one entry: the APK filename |
| `targetCommitSha` | String | The `headSha` from ReleaseTrigger |

**Validation rules**:
- `tagName` MUST NOT already exist as a git tag or GitHub Release (enforced by `softprops/action-gh-release`)
- `assets` list MUST contain exactly one entry
- `prerelease` is derived from `ReleaseTrigger.isPrerelease`

---

### VersionCommit

The git commit that persists the incremented `version.properties` back to `main`.

| Field | Type | Description |
|-------|------|-------------|
| `message` | String | `chore: bump version to {versionName} (code {versionCode}) [skip ci]` |
| `changedFile` | String | `version.properties` |
| `branch` | String | `main` |
| `authorName` | String | `github-actions[bot]` |
| `authorEmail` | String | `github-actions[bot]@users.noreply.github.com` |

**Notes**:
- The `[skip ci]` token in the commit message prevents `android-ci.yml` from triggering on this commit, which would otherwise cause a `workflow_run` re-trigger and a duplicate release attempt.
- If branch protection prevents bot push to `main`, this step is skipped and the fallback tag format `v{versionName}-{versionCode}` is used to ensure uniqueness without commit-back.

## Pipeline State Machine

```
[Trigger received]
      │
      ▼
[Preflight: verify Android SDK]
      │  FAIL → pipeline fails, no release created
      │
      ▼
[verify release hardening]
      │  FAIL → pipeline fails, no release created
      │
      ▼
[assembleRelease → increments version.properties → produces APK]
      │  FAIL → pipeline fails, no release created
      │
      ▼
[Read post-build version.properties → derive tag + APK path]
      │
      ▼
[Verify APK exists and size > 0]
      │  FAIL → pipeline fails, no release created
      │
      ▼
[Publish GitHubRelease with APK asset]
      │  FAIL (duplicate tag or permission error) → pipeline fails
      │
      ▼
[Commit updated version.properties to main with [skip ci]]
      │  WARN: if branch protection blocks push, log warning and continue
      │
      ▼
[Release published ✓]
```

## What This Feature Does NOT Introduce

- No new Room/database entities
- No new Kotlin data classes in the Android app modules
- No changes to the in-app data layer or presentation layer
- No new Gradle modules
- No changes to `NdiDatabase.kt`, `NdiRepositories.kt`, or any feature module
